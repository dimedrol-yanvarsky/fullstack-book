\subsection{Движок рендеринга Blink}

Blink~--- движок рендеринга, отвечающий за превращение HTML, CSS и JavaScript
в пиксели на экране пользователя. Ниже описан каждый этап конвейера рендеринга.

\subsubsection{HTML parser}

HTML-парсер получает поток байтов от сетевого модуля и преобразует его
в дерево DOM-узлов. Парсинг выполняется инкрементально: браузер не ждёт
полной загрузки документа, а начинает построение дерева по мере поступления
данных.

Когда парсер встречает тег \inl{<script>} без атрибутов \inl{async}
или \inl{defer}, он приостанавливает работу и передаёт управление
движку V8 для выполнения скрипта. Это объясняет, почему скрипты
рекомендуется размещать в конце \inl{<body>} или снабжать атрибутом
\inl{defer}.

\subsubsection{CSS parser}

CSS-парсер обрабатывает все подключённые стили (\inl{<link>},
\inl{<style>}, инлайновые атрибуты \inl{style}) и строит
\term{объектную модель CSS}{CSSOM}~--- древовидную структуру,
в которой каждый узел содержит набор вычисленных стилей.

CSSOM строится параллельно с DOM, однако рендеринг блокируется
до тех пор, пока CSSOM не будет полностью готов. Именно поэтому
CSS считается \term{блокирующим ресурсом рендеринга}{render-blocking resource}.

\subsubsection{DOM tree}

\term{Объектная модель документа}{DOM}~--- это древовидное представление
HTML-документа в памяти. Каждый HTML-элемент, текстовый узел, комментарий
и атрибут становится объектом с набором свойств и методов.

DOM является основным интерфейсом для JavaScript: через него скрипты
читают и изменяют содержимое страницы. Связь между DOM и движком V8
осуществляется через механизм привязок (Bindings), показанный
на рис.~\ref{fig:chromium-arch}.

\subsubsection{CSSOM}

CSSOM хранит результат каскадирования, наследования и вычисления
финальных значений CSS-свойств для каждого элемента. На этом этапе
относительные единицы (\inl{em}, \inl{\%}, \inl{vh}) пересчитываются
в абсолютные значения.

\important{Изменение стилей через JavaScript (например, присвоение
\inl{element.style.width}) вызывает пересчёт CSSOM и может
спровоцировать повторное прохождение последующих этапов конвейера~---
Layout, Paint и Compositing.}

\subsubsection{Layout tree}

На этапе Layout движок объединяет DOM и CSSOM и вычисляет геометрию
каждого видимого элемента: его позицию, размеры, отступы и границы.
Результатом является \term{дерево макета}{Layout tree}, которое
содержит только видимые элементы (элементы с \inl{display: none}
в него не попадают).

Операция Layout является рекурсивной: изменение размера родительского
элемента вызывает пересчёт всех его потомков. Стоимость полного
пересчёта составляет $O(n)$, где $n$~--- число узлов дерева.

\note{Частая причина проблем с производительностью~--- так называемый
\term{принудительный пересчёт макета}{forced reflow}: чтение
геометрического свойства (например, \inl{offsetHeight}) сразу после
изменения стилей заставляет браузер синхронно выполнить Layout.}

\subsubsection{Paint}

На этапе Paint браузер обходит Layout tree и генерирует список
графических команд (display list) для каждого элемента: залить
прямоугольник цветом, нарисовать рамку, отрендерить текст, наложить
тень и т.\,д. Порядок отрисовки определяется правилами
\term{контекста наложения}{stacking context} и свойством \inl{z-index}.

Сам по себе этап Paint не выводит пиксели на экран~--- он лишь
записывает инструкции, которые будут выполнены на следующем этапе.

\subsubsection{Compositing}

Современные браузеры разбивают страницу на независимые слои
(\term{композитные слои}{composite layers}). Каждый слой растеризуется
отдельно, часто с помощью GPU, а затем слои совмещаются в финальное
изображение.

Элементы выносятся в отдельный слой при наличии свойств
\inl{transform}, \inl{opacity}, \inl{will-change}, а также
при использовании \inl{<video>}, \inl{<canvas>} и CSS-анимаций.

\important{Избыточное количество слоёв (layer explosion) увеличивает
потребление памяти. Используйте \inl{will-change} осознанно
и только для элементов, которые действительно будут анимироваться.}

Compositing передаёт подготовленные текстуры слоёв процессу GPU,
который выполняет финальный рендеринг кадра и выводит результат
на экран пользователя.

\section{Работа движков Blink и V8 браузера Chromium на примере HTML-кода.}

Ниже представлен HTML-код страницы, содержащий CSS-селекторы и простой скрипт.
\begin{codelist}{html}{HTML-страница с CSS и JavaScript}
<!DOCTYPE html>
<html>
<head>
  <style>
    .container { width: 400px; padding: 20px; background: #f0f0f0; }
    #greeting { color: blue; font-size: 24px; }
  </style>
</head>
<body>
  <div class="container">
    <p id="greeting">Привет!</p>
    <button id="btn">Нажми</button>
  </div>
  <script>
    const btn = document.getElementById('btn');
    const greeting = document.getElementById('greeting');
    btn.addEventListener('click', () => {
      greeting.textContent = 'Нажато!';
      greeting.style.color = 'red';
    });
  </script>
</body>
</html>
\end{codelist}
